# 1. AI
## 개념
- 컴퓨터나 기계가 사람이 하는 것처럼 생각하고 행동하도록 하는 기술
- 기계가 스스로 생각하고 판단하여 행동하도록 하는 것

## 키워드
- 머신러닝 : 입력 받은 데이터를 분석하여 패턴을 인식함으로써 기계 스스로 예측 / 기계의 패턴인식 반복을 통해 자신의 오류를 수정하고 반복하여 정확도 상승
- 딥러닝 : 인공신경망에서 여러 계층을 쌓아서 만든 깊은 신경망 / 인공지능을 만드는 머신러닝의 한 종류
    - 머신러닝은 인간이 특징을 추출할 필요 / 딥러닝은 인간의 개입 없이 심층 신경망을 통해 스스로 학습
- 지도학습 : 입력값과 결과값을 주고 함께 학습시키는 방법
- 비지도학습 : 입력값만을 주고 학습시키는 방법

## 장점
- 인간의 단순 노동을 대체
- 업무의 효율성과 편리성 증대
- 예측 및 방지를 통한 서비스의 효율성 증대

## 단점
- 실업자 증가
- 윤리적 문제 발생
- 방대한 데이터 수집으로 인한 개인정보 오남용
- AI 기업이 독점하여 빅브라더 문제
- 부적절한 데이터 습득 시 잘못된 AI 개발이 가능

## 활용
- 데이터를 분석 및 시각화
- AI 분석 모델을 기반으로 해결 방안 제시
- 부정행위 등 이상 상황에 대한 감지
- 악성코드, 웹 해킹 등 사이버 공격을 자동으로 탐지 및 분석하고 공격 차단
- 개인의 취향과 기호, 날씨 등을 바탕으로 맞춤형 메뉴를 추천
- 대부분의 딥러닝 기반 탐지기는 원본 데이터를 확인 후, 비진위성의 징후를 찾고 영상의 문제를 식별

# 2. 빅데이터
- 기존의 처리 방법으로는 감당하기 힘들 정도로 큰 데이터
- 쌓은 경험이나 지식을 바탕으로 미래를 예측
- 단순 수치를 통한 미래 예측이 중요한 것이 아니라 미래 구상이 중요
- 수집한 데이터가 매우 중요하므로 해당 데이터에 대한 신뢰도가 빅데이터를 다룰 때의 핵심

## 키워드
- 정형 데이터 : 형식과 구조가 존재하는 데이터
- 비정형 데이터 : 정의된 구조가 없으며 연산이 불가능한 데이터 / 
    - 어디에 활용될 데이터인지 정확한 분류 X / 영상, 텍스트, 위치, 숫자 등 다양한 형태 / 복잡도가 높아 기존에 사용하는 성능이나 기술로는 처리 불가
    - 정형 데이터로 변환하는 전처리 과정 필요
- NoSQL : 비관계형으로 데이터를 구조화하는 데이터베이스

## 장점
- 미래 예측
- 문제 파악 등의 방식으로 환경 개선

## 단점
- 무분별한 개인정보 수집
- 미래 예측에 대한 보장 X
- 수집한 데이터에 대한 신뢰도 문제

## 활용
- 빅데이터 재난예측시스템은 특정지역에 유동인구가 밀집되는 것을 감지하고 인공지능을 활용해 단계별로 위기상황을 전달하는 시스템
- 사용자의 실시간 행동, 인터넷 검색 패턴, 구매 상품에 대한 반응, 장바구니 내역과 같은 데이터를 수집하여 개인화 서비스 제공
- 서비스가 즉각적으로 사용자를 알아보고 기억하고 연관된 제품을 제안
- SNS 등 개인의 다양한 빅데이터를 수집해 긍정적인 요소를 바탕으로 한 신용평가 서비스
- 넷플릭스는 신작 VOD 제공이 아닌 사용자가 좋아하고 오래볼 수 있는 VOD를 제공
- Haven의 헬스케어 분야 - 생성되는 기존 데이터의 표준화 필요 / 변환의 어려움 / 사용자의 낮은 참여도로 인해 실패

# 3. IoT
- 각종 사물에 센서와 통신 기능을 내장하여 인터넷에 연결하는 기술
- Machine-to-Machine의 3가지 기술
    - 네트워킹 기술 : 인간, 사물, 서비스 등 분산된 요소들을 서로 연결하는 유무선 통신 시스템
    - 센싱 기술 : 센서를 통해 원격감지, 위치 및 모션 추적 등의 방식으로 사물과 주위 환경으로부터 정보를 획득하는 기술
    - 인터페이스 기술 : 사물인터넷의 특정 기능을 수행하는 응용서비스와 연동시키는 기술

## 장점
- 실시간 모니터링 및 외부 제어를 통한 편의성 증대
- 수집한 정보를 통해 실시간으로 최적의 대응 가능

## 단점
- 기기의 결함으로 인한 오작동이나 주요 정보 노출
- 프라이버시 보안 이슈

## 활용
- 고객 경험 향상을 위해 센서를 활용하여 고객 행동, 서비스 접근 등의 데이터를 수집
- 스마트폰으로 오프라인 상점에서 결제뿐만 아니라 ATM 이용이나 NFC 태그를 활용하여 예약한 호텔에서 객실 키로 사용 가능
- 전세계에 널리 분포된 IoT 장비를 원격으로 관리
- IoT센서에 발열, 누전 등 이상신호를 감지 시 경보알림 기능, 관리자가 케이블 상태를 웹과 모바일을 통해 실시간으로 확인 가능

# 4. 블록체인
- 분산저장된 단순한 회계 장부가 아닌, 네트워크를 기반으로 한 분산 데이터 구조
- 메커니즘
    - ① 일정 시간 동안 발생한 모든 거래정보를 블록(block) 단위로 기록하여 
    - ② 이를 모든 구성원들에게 전송하고 
    - ③ 블록의 유효성이 확보될 경우 
    - ④ 기존의 블록에 추가로 연결(chain)하여 보관

## 특징
- 중앙 서버를 거치지 않고 클라이언트끼리 직접 통신하는 P2P(Peer-to-Peer) 방식
- 소규모 데이터가 사슬 형태로 무수히 연결되어 형성된 블록이라는 분산 데이터 저장 환경에 관리 대상 데이터를 저장
    - 누구도 수정 불가하고 누구나 변경의 결과를 확인 가능
- 데이터의 진위성 여부를 중앙집중의 인증방식을 통하지 않아도 해결 가능
- 블록체인의 보안 알고리즘은 서로 신뢰 불가능한 참가자 사이에서 동일한 데이터를 유지하는 것이 목표
- 데이터 부분과 해시 부분으로 구성
- 데이터를 담은 블록과 해시를 이용한 연결이며 연결된 블록은 네트워크를 통해 수많은 컴퓨터에 복제되어 저장

## 키워드
- 프라이빗 블록체인 : 허가된 사용자만 참가 가능한 블록체인 네트워크
- 51% 어택 : 과반수의 참여자가 동시에 블록체인 거래내역을 조작하는 행위
- 더블 스펜딩 : 동일한 암호화폐로 두 번 이상 지출
    - 트랜잭션이 충돌 시 더 많은 노드가 받은 트랜잭션을 통과시키고 다른 트랜잭션은 취소되는 특징을 이용
    - 거래의 승인 수가 일정 수치 넘었는지를 확인하여 해결

## 장점
- 분산형 네트워크를 통해 정보 및 가치를 저장하고 관리함으로써 신뢰성을 보장
- 조작을 하려면 연결된 모든 개인용 블록을 신규 블록 생성 이전에 조작해야 하므로 사실상 불가능하므로 불변성을 보장
- 위조 및 변조가 불가능하므로 투명성을 보장

## 단점
- 블록체인 거래가 많아지면 블록에 저장되는 정보의 용량도 커져 확장성의 문제가 발생하여 다시 중앙 집중화 초래 가능
- 데이터의 무결성과 성능을 위해 10분마다 블록이 생성되지만 분기를 방지하기 위해 6블록이 기준이므로 1시간이 걸리기에 대량 데이터의 일괄 처리나 실시간 이벤트 처리에는 적합 X 
- 문제 발생 시 책임 소재가 모호

## 활용
- 중앙집중 방식으로 수십억대의 IoT 디바이스를 연결해 서비스를 관리하는 것은 비용, 안정성에서 안 좋음
- IoT와 블록체인 활용은 크게 사물 간 지급 결제, 사물 보안, 자동화 과정 처리
- 보안, 안전 등의 이슈를 중재자에 의존하는 중앙집중 구조는 비용과 신뢰에서 안 좋음 / 블록체인을 통해 신뢰를 분산시켜 중재자 없이
직거래를 하고 임의로 조작 못하는 플랫폼을 제공하며 시스템 간의 데이터 동기화 필요성을 제거하여 시스템 구조의 효율성 증대
- 위조 및 변조가 불가능한 특징을 살려 제품의 구성이나 유통 과정 등을 투명하게 공개

# 5. 클라우드
- 인터넷의 가상 서버에 프로그램을 두고 필요할 때마다 기기를 통해 불러 사용하는 서비스

## 키워드
- 가상화 : 단일 물리 H/W 시스템에서 시뮬레이션 환경이나 전용 리소스를 생성하는 기술
- 온프레미스 : 데이터 센터나 서버실에 서버를 두고 직접 관리하는 방식 / IT 자원을 사용자가 직접 관리
- 로드밸런싱 : 서버가 받는 부하를 분산해주는 기술
- 오토스케일링 : CPU, 메모리 등의 자원을 모니터링하여 서버 사이즈를 자동으로 조절하는 기술
- 스케일업 : 서버 자체의 성능을 확장하는 방식
- 스케일아웃 : 기존 서버와 동일하거나 낮은 성능의 서버를 추가하여 운용하는 방식

## 장점
- 투명성 상승 및 장기적 비용 절감
- 확장성
- 자원에 대해 유연하고 탄력적인 인프라 운영 가능
- 종량제 형식이므로 합리적인 비용
- 서버에 부하가 발생하지 않도록 신속한 증설 가능
- 클라우드가 데이터 보관, 인프라 관리 등의 작업을 맡고 기업은 온전히 비즈니스에 집중하도록 지원 

## 단점
- 전문인력 필요
- 관리 부실 시 자료 유출 및 해킹 등의 보안 이슈
- 서비스를 많이 이용할수록 클라우드 업체에 대한 의존도/종속성 상승
- 기업의 핵심 데이터를 외부에 보관한다는 불안감

## 클라우드 서비스의 종류
- Iaas(Infra)
    - 사용자가 관리할 수 있는 범위가 가장 넓은 클라우드 서비스
    - 사용한 시간만큼 비용 발생
    - AWS, Azure
- Paas(Platform)
    - 가상화된 클라우드에 개발 환경을 미리 구축하여 사용자가 원하는 서비스를 개발할 수 있도록 하는 클라우드 서비스
    - 사용한 메모리 양만큼 비용 발생
    - 사용자는 애플리케이션 개발과 비즈니스에만 집중 가능
    - Heroku
- Saas(Software)
    - 클라우드 인프라 위에 소프트웨어를 탑재해 제공하는 클라우드 서비스
    - 스토리지 수용량만큼 비용 발생
    - 자원뿐만 아니라 소프트웨어의 업데이트, 버그 개선 등의 서비스를 업체가 맡아서 제공
    - Slack, Dropbox

## 클라우드 분류
- Private Cloud
    - 특정 다수에게 서비스 형태로 제공되는 시스템
    - 기업 내에서만 사용되는 서비스
    - 잠시라도 끊기면 안되는 시스템 / 클라우드 서비스 제공자가 보장하는 것 이상의 가용성이 필요 / 기밀성이 필요할 때 유리
- Public Cloud
    - 불특정 다수에게 서비스 형태로 제공되는 시스템
    - 리소스를 직접 관리할 필요 X / 온프레미스 환경보다 신속
- Hybrid Cloud
    - 점진적인 클라우드 마이그레이션에 용이
    - 이벤트와 같이 트래픽이 집중되는 웹 서버는 퍼블릭 클라우드에, 고객 데이터와 같은 핵심 데이터는 온프레미스에서 운영하여 전용 네트워크로 연결
    - 보안과 안정성 우수
- Multi Cloud
    - 서로 다른 업체에서 2개 이상의 퍼블릭-퍼블릭 or 프라이빗-프라이빗 클라우드를 이용해 하나의 서비스를 운영하는 것
    - 업체의 종속성을 피하고 한 시스템에 장애가 발생 시 다른 시스템을 이용함으로써 타격 X
    - 연결되지 않고 개별적으로 사용한다는 점에서 하이브리드 클라우드와 차이

## 활용
- 데이터 저장 솔루션으로 멀티 클라우드를 택하여 데이터 저장 인프라를 대체
- 더이상 단일 퍼블릭 클라우드는 보안적 측면에서 선택 X
